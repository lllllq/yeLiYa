正向代理和反向代理的区别
正向代理一般就是客户端向服务器端发送请求的时候中间会有一个代理服务器，先由代理服务器接收请求，然后代理服务器将请求发送给真正的服
务器。那么问题来了，为什么中间要有一个代理服务器呢？答案就在于有些时候客户端不能直接访问服务器。现实中的例子就是翻墙。不过自从VPN技术被广泛应用外，“翻墙”不但使用了传统的正向代理技术，有的还使用了VPN技术。还有可能是为了能加速访问服务器。使用正向代理时客户端必须设置正向代理服务器，当然前提是知道代理服务器的ip地址还有代理程序的端口。

反向代理也是客户端向代理服务器发送请求，然后由代理服务器向服务器发送请求。只不过使用反向代理时并不需要设置代理服务器，对于客户端而言代理服务器就像真正的服务器一样。

以下为nginx中server的主要配置
server {
	keepalive_requests 120; #单连接请求上限次数。
	listen       4545;   #监听端口
	server_name  127.0.0.1;   #监听地址       
	location  ~*^.+$ {       #请求的url过滤，正则匹配，~为区分大小写，~*为不区分大小写。
	   #root path;  #根目录
	   #index vv.txt;  #设置默认页
	   proxy_pass  http://mysvr;  #请求转向mysvr 定义的服务器列表
	   deny 127.0.0.1;  #拒绝的ip
	   allow 172.18.5.54; #允许的ip           
	} 
}
