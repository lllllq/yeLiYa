fis3在我的理解中就是一个对前端代码进行管理的工具。我看别人都叫他前端构建工具，那我也这样叫吧。在桥隧的项目中我们安装了nodejs和npm等。其实现在想想对于node我们在项目中并没有实际用到。只是用它来安装npm(包管理工具),然后再使用npm来安装fis3。安装方式：npm install -g fis3.

fis3在构建的过程中并不会改变源码，他只是把源码构建之后放到指定的目录下。例如：fis3 release -d E:\publicCompany\nginx\html会将当前项目构建之后放到e盘这个目录下。

那构建过程又是什么呢？主要还是围绕前端开发所需要的三种编译能力：1资源定位 2内容嵌入 3依赖声明。这三种方法具体是什么东西，等我以后再研究

配置文件fis-config.js
编译的整个流程都是根据配置来的，使用了类似css的配置方式，固化了配置过程，简化了流程。复杂的配置我就不写了，网上有

调试
fis3有一个内置的server，当不指定发布的地址时，默认发布到这个server中(fis3 release)，打开方式：fis3 server open可以打开此目录
然后使用fis3 server start命令启动项目，当启动后，会自动打开浏览器127.0.0.1：8080。默认监听端口号8080，也可以更改。
